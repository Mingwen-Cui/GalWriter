# GalWriter AI ユーザーガイド

<p align="center">
  <img src="./public/icon.png" alt="GalWriter AI" width="96" />
</p>

GalWriter AI は、ビジュアルノベル、ギャルゲー、分岐型ストーリー、マルチルートのシナリオ制作向けのインタラクティブエディタです。シナリオノード、キャラクターやシーンの資料、AI による続き生成、画像生成、音声読み上げ、リアルタイムテスト、エクスポートを 1 つのキャンバスで扱えるので、複雑な物語の整理、分岐案の素早い試作、素材管理、後工程への受け渡しがしやすくなります。

このドキュメントは利用者向けです。プロジェクトの作成、API 設定、API テンプレートの取り込み、執筆中の AI 利用方法を説明します。

## はじめに

初めて GalWriter AI を使う場合は、次の手順で始められます。

1. アプリを開き、プロジェクトホームで **新規作成** を押すか、既存プロジェクトを読み込みます。
2. キャンバスにシナリオ、キャラクター、シーン、背景などのノードを追加し、分岐構造をつなぎます。
3. 右側の **設定** で Text AI、Image AI、Voice AI を設定します。まだ AI を使わない場合は後回しでも構いません。
4. 生成が必要になったら **AI アシスタント** を開き、AI 続き生成を使うか、該当ノードから画像・音声機能を呼び出します。
5. 作業が終わったらプロジェクトを保存し、必要に応じてエクスポートします。

補足:

- 画面構成を先に知りたい場合は、下の **画面の見方** を参照してください。
- API 設定をしたい場合は、**API 設定**, **Text AI API**, **Image AI API の取り込み**, **Voice AI API** を参照してください。

## ローカル開発とビルド

### 依存関係をインストール

```bash
npm install
```

### Web アプリをローカルで起動

```bash
npm run dev
```

起動後、ブラウザで `http://localhost:3000` を開いてください。

### デスクトップアプリを開発モードで起動

```bash
npm run tauri dev
```

フロントエンドの開発サーバーを起動し、Tauri のデスクトップウィンドウを開きます。

### Web をビルド

```bash
npm run build
```

ビルド結果は `dist/` に出力されます。`npm run preview` で静的ビルドをローカル確認できます。

### デスクトップアプリをビルド

```bash
npm run tauri build
```

フロントエンドのビルドを実行した後、デスクトップ用インストーラーとプラットフォーム別の成果物を生成します。

### 静的ビルドをプレビュー

```bash
npm run preview
```

`dist/` の内容をローカルで確認するために使います。

デスクトップアプリの実行やパッケージ作成には、Rust と Tauri の実行環境が必要です。ブラウザ版だけを使う場合は、Node.js と npm だけで十分です。

## 画面の見方

- **プロジェクトホーム**: 新規作成、既存プロジェクトの読み込み、最近使ったプロジェクトのオープン、まとめてダウンロード/削除ができます。
- **中央キャンバス**: シナリオ、キャラクター、シーン、背景、条件などのノードを配置して接続します。
- **左ツールバー**: さまざまなノードや素材を追加します。
- **右ツールバー**: 設定、タイトル表示の切り替え、元に戻す/やり直し、キャンバス背景の変更、AI アシスタントの起動ができます。
- **AI アシスタント**: AI と会話し、参考資料をアップロードし、生成した内容を現在の制作に反映できます。
- **設定パネル**: AI 接続、キャンバス外観、プロンプトテンプレート、AI 続き生成ボタンなどを設定します。

## 初回利用

1. アプリを開き、プロジェクトホームで **新規作成** を押します。
2. キャンバスにシナリオ、キャラクター、シーン、背景のノードを追加します。
3. ノードを接続して分岐構造を作ります。
4. AI を使う場合は、先に **設定** で API 設定を完了します。
5. プロジェクトを保存し、後から **最近使ったプロジェクト** から再度開けます。

## API 設定

1. エディタ右側のツールバーで **設定** をクリックします。
2. **AI API 設定** セクションを探します。
3. 設定したい種類を選びます。
   - **Text AI**: 続き生成、推敲、分析、会話、AI アシスタント向け。
   - **Image AI**: キャラクター、シーン、背景画像の生成向け。
   - **Voice AI**: 音声合成や読み上げ向け。
4. 対応する欄の右上にある **新規設定** をクリックします。
5. 設定名、Provider、Model、API Key、API URL を入力します。
6. **保存** をクリックします。
7. 設定一覧に戻り、保存した設定をクリックして有効化します。

AI 設定は現在の端末にのみ保存され、プロジェクトと一緒には書き出されません。別の端末では再入力が必要です。

## Text AI API

Text AI は、AI 続き生成、アイデア出し、リライト、シナリオ挿入、情景描写、会話補完、AI アシスタントなど、多くの執筆機能で使われます。

代表例:

| Provider | API URL 例 | Model 例 |
| --- | --- | --- |
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` / `deepseek-reasoner` |
| Gemini | `https://generativelanguage.googleapis.com` | `gemini-2.5-flash` |
| OpenAI | `https://api.openai.com/v1` | `gpt-4.1` |
| Claude | `https://api.anthropic.com/v1/messages` | `claude-sonnet-4-20250514` |
| Kimi | `https://api.moonshot.cn/v1` | `kimi-k2.5` |
| 通義千問 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen3.6-plus` |
| GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-5` |
| カスタム | 提供元のエンドポイントを入力 | 提供元のモデル名を入力 |

入力の目安:

- **API Key**: プロバイダの管理画面で発行したキーを貼り付けます。
- **API URL**: まずは初期値を使い、プロキシや互換エンドポイント、中継サービスを使う場合はその URL に変更します。
- **Model**: ドロップダウンから選ぶか、カスタム名を手入力できます。
- **思考モード**: `deepseek-reasoner` のような推論出力対応モデル向けです。通常のチャットモデルではオフにします。

保存後、Text AI の設定は AI 続き生成、AI アシスタント、AI 集約分析で使われます。

## Image AI API の取り込み

Image AI 設定画面では、クリップボードからテンプレートを自動取り込みできます。プロバイダのドキュメントから curl、JSON、公式サンプルをコピーする場合に特に便利です。

手順:

1. プロバイダの管理画面や公式ドキュメントから画像生成のサンプルをコピーします。
2. サンプルにはできれば次の項目を含めます。
   - `api_key` / `apiKey` / `OPENAI_API_KEY` / `ARK_API_KEY`
   - `base_url` / `baseURL` / `api_url` / `endpoint`
   - `model`
   - `size`
3. GalWriter AI に戻ります。
4. **設定 > AI API 設定 > Image AI > 新規設定** を開きます。
5. Provider を選ぶか、**カスタム** を選択します。
6. **テンプレートを取り込み** をクリックします。
7. クリップボードから API URL、API Key、Model、サイズを読み取って入力します。
8. 結果を確認し、必要なら手動で修正します。
9. **保存** をクリックし、Image AI の一覧で保存した設定を選びます。

取り込み例:

```bash
curl https://api.openai.com/v1/images/generations \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-1",
    "prompt": "a visual novel background",
    "size": "1024x1024"
  }'
```

または:

```json
{
  "api_key": "YOUR_API_KEY",
  "base_url": "https://api.example.com/v1/images/generations",
  "model": "example-image-model",
  "size": "1024x1024"
}
```

取り込み時に有効な項目が見つからない場合は、次の項目を手動で入力できます。

- **API URL**: 画像生成のエンドポイント。
- **API Key**: プロバイダのキー。
- **Model**: 画像モデル名。
- **Size**: `1024x1024` などの出力サイズ。

## Voice AI API

Voice AI はテキストを音声に変換します。OS 標準の音声も、オンライン音声サービスも使えます。

- **システム音声**: デスクトップアプリ内蔵の読み上げ機能を使います。API Key は不要です。
- **有道音声**: App Key と App Secret を入力します。
- **OpenAI 音声**: API Key、API URL、Model、Voice を入力します。
- **豆包 / Gemini / カスタム音声**: 各サービスの要件に従って API URL、API Key、Model、Voice を入力します。

Voice は音声名です。たとえば `alloy`、`Kore`、またはサービスが提供する中国語音声 ID などです。

## API が有効か確認する方法

設定を保存して選択したら、次の方法でテストできます。

- Text AI: シナリオノードを選び、AI 続き生成を押すか、AI アシスタントでテストメッセージを送ります。
- Image AI: キャラクター、シーン、背景関連の機能で画像生成を試します。
- Voice AI: 読み上げ対応のテキストで音声生成または再生を試します。

失敗した場合は、まず次を確認してください。

- API Key に余分な空白が入っていないか。
- API URL が正しいエンドポイント階層になっているか。
- Model 名が提供元ドキュメントと一致しているか。
- 使いたい設定が有効になっているか。
- アカウントに残高、利用枠、モデル権限があるか。
- ネットワークからそのサービスにアクセスできるか。

## プロジェクト管理

### プロジェクトの作成と読み込み

- **新規作成** を押すと空のキャンバスから始められます。
- 最近使ったプロジェクトのカードをクリックすると開けます。
- プロジェクトカード右側のボタンから、個別にダウンロード、名前変更、削除ができます。

### 一括操作

プロジェクトホーム左側の **一括選択** を押すと、複数の最近使ったプロジェクトを選択できます。

- **ダウンロード**: 選択したプロジェクトを順番に書き出します。
- **削除**: 選択したプロジェクトをまとめて削除します。削除前に確認があります。

### インポートとエクスポート

- **既存プロジェクトを読み込み**: 以前に書き出したプロジェクトファイルを選んで再読み込みします。
- **ダウンロード / プロジェクトを書き出す**: プロジェクト構造と関連素材をエクスポートし、バックアップ、移行、受け渡しに使えます。

## 基本的な制作フロー

1. シナリオノードを追加し、本文、選択肢、分岐内容を書きます。
2. キャラクター、シーン、背景ノードを追加して、素材と設定をまとめます。
3. 矢印でノードをつなぎ、ストーリーの流れを作ります。
4. AI 続き生成、推敲、分析を使って内容を素早く補完します。
5. リアルタイムテストでプレイヤー視点の分岐体験を確認します。
6. 必要に応じて保存し、エクスポートします。

## AI 執筆機能

Text AI を設定すると、次の機能が使えます。

- **前文を引き継いで続き生成**: 既存のシナリオから続きを書きます。
- **創作分岐案の生成**: 複数の展開案を出します。
- **極上の推敲・リライト**: 情報は保ったまま表現を整えます。
- **シナリオ挿入**: 既存の章の間に橋渡し文を追加します。
- **情景描写**: 雰囲気、空気感、カメラ感を強めます。
- **会話補完**: キャラクターと文脈に応じて台詞を補います。
- **AI 集約分析ノード**: 複数ノードをつないで、プロットの整合性、人物状態、文脈を要約します。

設定画面では **AI 続き生成ボタン** も調整でき、よく使う操作だけを残せます。

## カスタムプロンプト

設定パネルで AI プロンプトテンプレートを編集できます。次のような場面に便利です。

- AI に固定の文体を使わせたい。
- キャラクター設定をより重視させたい。
- 続き生成、推敲、シナリオ挿入の出力形式を変えたい。
- 自分の制作フローやチームのルールに合わせたい。

プロンプトは現在のプロジェクトに保存されます。プロジェクトごとに別のプロンプトを使えます。

## よくある質問

### API Key を入れたのに AI が使えません

設定を保存し、設定一覧で選択済みか確認してください。保存していない下書きは有効になりません。

### API URL はベース URL ですか、それとも完全なエンドポイントですか

プロバイダによります。まずはアプリが自動入力した値を使ってください。カスタムや中継サービスを使う場合は、提供元のドキュメントに従ってください。画像生成は、完全な画像エンドポイントが必要になることが多いです。

### 画像 API テンプレートの取り込みでクリップボード内容は保存されますか

いいえ。テンプレート取り込みは現在のクリップボード文字列を読み取って項目を推定するだけです。結果は現在の設定フォームに入るだけで、保存を押したときに初めてローカル保存されます。

### プロジェクトの書き出しに API Key は含まれますか

いいえ。AI API 設定は現在の端末にのみ保存され、プロジェクトとは一緒に書き出されません。これにより秘密情報がプロジェクトファイルに混ざりません。

### 複数の API を設定できますか

できます。Text、Image、Voice それぞれで複数の設定を保存できます。切り替えるときは、設定一覧で使いたいものをクリックしてください。

## ローカル Stable Diffusion WebUI

Image AI の Provider には **Local Stable Diffusion WebUI** を選べます。ローカルの AUTOMATIC1111 WebUI に接続するための設定です。

1. Stable Diffusion WebUI を API 有効で起動します。たとえば起動オプションに `--api` を追加します。
2. ブラウザやデスクトップアプリで CORS エラーが出る場合は、`--cors-allow-origins=*` を追加するか、現在のアプリ URL だけを許可します。
3. アプリで **設定 > AI API 設定 > Image AI > 新規設定** を開きます。
4. Provider を **Local Stable Diffusion WebUI** にします。
5. API URL は通常 `http://127.0.0.1:7860` です。完全なエンドポイント `http://127.0.0.1:7860/sdapi/v1/txt2img` を直接入れても構いません。
6. API Key は通常空欄のままで大丈夫です。
7. Negative prompt、Steps、CFG Scale、Sampler、Seed、Restore Faces、Hires Fix などを調整できます。

この Provider は `POST /sdapi/v1/txt2img` を呼び出し、返ってきた base64 画像をシナリオカード、キャラクターカード、シーンカードに直接書き込みます。
