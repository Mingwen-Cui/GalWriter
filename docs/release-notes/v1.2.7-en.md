# GalWriter AI v1.2.7

> Compared with v1.2.5, this release is more than a small patch set. It upgrades project saving, AI profile management, mobile layout work, Android packaging, visual polish, and several key node behaviors, while still fixing the save button falsely lighting up after opening a project and other state-sync issues.

**[中文版 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-zh.md) | [日本語 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-ja.md)**

## Overview

GalWriter AI v1.2.7 is a broader release aimed at formal launch readiness. Compared with the v1.2.5 base of local projects, AI profiles, and video export, this version continues to expand mobile layout work, Android support, AI profile restructuring, API guidance, background-removal integration, visual consistency, and project state management, while also moving multimedia export away from the older external-FFmpeg packaging model toward a lighter, more integrated approach.

The biggest change is a re-alignment of project snapshots, manual saves, autosave, and recovery, combined with the main user-facing improvements accumulated after v1.2.5. After opening an existing project, the save button should no longer appear dirty by default; when nothing has changed, autosave and recovery prompts should stay quiet as well. Project title, display name, and save file name responsibilities were also clarified so the editor no longer feels like it changed something just by loading a project.

---

## Highlights

- **No false dirty state on project open**: opening an existing project should no longer make the save button look active immediately.
- **More stable autosave behavior**: autosave now uses the same snapshot baseline as manual save, reducing mismatched state.
- **Less intrusive recovery prompts**: recovery is only shown when an unsaved autosave draft actually exists.
- **Clearer title / display-name behavior**: project title is no longer polluted by the file name, so the editor load state is easier to trust.
- **More complete close flow**: on desktop, quitting now respects unsaved changes before closing.
- **More consistent save / restore flow**: serialization, restoration, and save-back paths were tightened together.
- **Lighter multimedia export**: the desktop export flow moved from an external FFmpeg bundle to a more integrated approach, making the release package simpler.
- **Stronger AI profile center**: text, image, background-removal, and voice provider profiles are organized more consistently.
- **Better mobile and Android readiness**: layouts and interactions were pushed further for phone screens, and Android packaging is part of the delivered platform set.
- **Cleaner visual polish**: button states, theme colors, and several node visuals were updated together.
- **Updated API guidance**: background-removal and AI usage guidance were refreshed to match current support.

---

## Added

### More consistent project state synchronization

When a project opens, the editor now waits for the current snapshot to settle before treating it as the saved baseline. This avoids false dirty states caused by restore order, field order, or temporary state mismatch.

### Desktop close confirmation

When there are unsaved changes on desktop, the app now routes through an in-app confirmation flow first, reducing the chance of accidental data loss.

### Mobile and Android readiness

The editor and related panels received more responsive layout work for phone-sized screens, and the Android build path is now part of the supported delivery story.

### AI profile restructuring

AI provider profiles were reorganized so text, image, background-removal, and voice settings feel more unified and easier to manage across multiple providers.

### Integrated multimedia export

Desktop multimedia export no longer relies on the v1.2.5-style external FFmpeg bundle. It now uses a lighter, more integrated implementation, which reduces release size and simplifies the export path.

---

## Changed

### Save and autosave

Manual save and autosave now share a unified snapshot comparison model, which reduces the chance of a project remaining marked as dirty after saving.

### Project recovery

Recovery prompts are now shown more conservatively, so projects that were saved normally do not keep asking to restore previous progress.

### Project title handling

The relationship between project title, save file name, and display name has been cleaned up to avoid load-time state confusion.

### API and visual updates

Background-removal API guidance, related AI usage notes, and the overall UI color/interaction polish were updated to better match current product behavior.

### Multimedia export packaging cleanup

The desktop multimedia export flow was consolidated into a lighter package, so users no longer need to reason about a separate FFmpeg distribution model during normal use.

---

## Fixed

- Fixed the save button appearing active immediately after opening a project.
- Fixed autosave from firing when nothing had actually changed.
- Fixed projects asking to restore previous progress again after a normal save.
- Fixed title / file-name interactions that made a freshly opened project look modified.
- Fixed inconsistent state write-backs in parts of the save and restore flow.
- Fixed several mobile layout and phone-view issues.
- Fixed the emergency logic issue in the number-condition node.
- Fixed unwanted default empty AI profiles appearing in some states.
- Fixed the release package being heavier and more fragmented than necessary for multimedia export.

---

## Upgrade Notes

### Best suited for the public experience rollout

If you are upgrading from an older version, it is a good idea to open an existing project, save it once, and reopen it to confirm that the new snapshot baseline is aligned.

### Data compatibility

This release keeps a local-first compatibility posture. Existing projects should continue to open and save without manual migration.

### Major upgrades since v1.2.5

If you think of v1.2.5 as the base release, the main upgrades delivered by v1.2.7 can be summarized as:

- More complete AI profile management
- Better mobile UX
- Android packaging support
- Lighter integrated multimedia export
- More consistent visual polish
- More stable save / recovery behavior
- Updated API guidance and compatibility wording

## Known Limitations

- This release still targets Windows desktop use as the primary experience.
- Web video export remains limited by browser constraints.
- AI provider availability, quotas, billing, and policies still depend on the configured service.
- Local Stable Diffusion WebUI still needs to be started and exposed by the user.
