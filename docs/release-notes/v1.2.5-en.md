# GalWriter AI v1.2.5

> This release focuses on making GalWriter AI a better local creative workspace: projects can be saved locally, AI provider settings and API keys stay out of project exports, the desktop app gains a fuller video export path, and the web version keeps WebM as the stable direct export format while guiding MP4 / MOV / MKV users to the desktop app.

**[中文版 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.5-zh.md) | [日本語 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.5-ja.md)**

## Overview

GalWriter AI v1.2.5 moves the editor further from a temporary web tool toward a local project workspace that can support ongoing writing.

This release improves three major areas: local project management, safer AI provider configuration, and video export UX. Users can create, import, open, rename, and delete projects from the project list; AI settings are stored locally to prevent API keys from being bundled into project files; the desktop app now has Full / Lite installer variants, where Full bundles a slim FFmpeg build and Lite keeps the download smaller; the web version continues to use WebM as its stable direct export format.

---

## Highlights

- **Local project library**: create a new project, import a project, or continue from recent projects.
- **Safer API key storage**: AI provider settings are stored locally and are not exported with project ZIP files.
- **Project-scoped AI conversations**: AI conversations belong to the current project only.
- **Unified AI settings**: text, image, and voice AI can each keep multiple saved provider profiles.
- **Better video export guidance**: on the web, choosing MP4 / MOV / MKV now shows an in-app desktop download prompt instead of a native browser alert.
- **Desktop Full / Lite installers**: Full bundles a slim FFmpeg build and is recommended for most users; Lite excludes FFmpeg for smaller downloads.
- **Desktop FFmpeg detection**: the app checks bundled `ffmpeg.exe`, the app directory, then system PATH, and shows an in-app prompt if FFmpeg is missing.
- **More professional project docs**: Chinese, English, and Japanese README files have been rewritten for a clearer open-source landing page.

## Added

### Local Project Management

Added a project picker for creating, importing, opening, deleting, and renaming recent projects.

Users can continue from recent projects after entering the editor. If no project exists, they can create a new one or import an existing project. AI conversations, story nodes, and project settings are saved with the current project.

### AI Provider Configuration Center

Added unified AI provider profile management:

- **Text AI**: AI assistant, continuation, polishing, analysis, and other writing features.
- **Image AI**: character, scene, and background image generation.
- **Voice AI**: narration, dubbing, and text-to-speech.

Each type can store multiple profiles and select one active profile. Profile name, Provider, Model, API URL, and API Key are saved locally.

### Video Export Format Strategy

The desktop app can export common video formats through FFmpeg. The Full installer bundles `ffmpeg.exe`; the Lite installer does not. If users choose Lite, MP4, MOV, and MKV export requires FFmpeg to already be available on the computer. The web version only supports direct WebM export because of browser limitations.

When a web user selects MP4, MOV, or MKV, GalWriter AI now shows a consistent in-app prompt and links to the desktop release page.

### Desktop Installer Split

Added two desktop release assets:

- `GalWriter-Setup.exe`: Full build with a slim bundled FFmpeg, recommended for most users.
- `GalWriter-Setup-Lite.exe`: Lite build without bundled FFmpeg, smaller download; MP4 / MOV / MKV export requires system FFmpeg.
- `GalWriter-AI-v1.2.5-windows-x64.exe`: portable Lite build, no installation required, double-click to run.
- `GalWriter-AI-v1.2.5-windows-x64-portable-full.zip`: portable Full build; extract it, then run the app while keeping `ffmpeg.exe` in the same folder.

The build scripts only bundle `ffmpeg.exe` in the Full installer. They do not include `ffprobe.exe`, documentation, presets, or unrelated FFmpeg files.

## Changed

### Project Save and Recovery Logic

Fixed the counterintuitive behavior where a normally saved project could still ask whether to restore previous progress after reopening.

The recovery prompt now appears only when an unsaved autosave draft exists. If the project was saved normally, reopening it will not show the recovery prompt again.

### Project Title Editing

Project names are now edited from the editor header. If the title is empty when saved, the app generates a default project name automatically.

### Unified Modal Style

Delete confirmations, AI conversation deletion, and web video format prompts now use in-app UI modals wherever possible instead of browser-native alert / confirm dialogs.

### Recent Project Scrolling

When there are many recent projects, the project window scrolls internally instead of stretching the whole modal.

## Fixed

- Fixed a race where a pending autosave timer could write an old draft after a successful save.
- Fixed false recovery prompts after closing or reopening a web project.
- Fixed the project picker closing unexpectedly.
- Fixed header layout issues when project names are long.
- Fixed unwanted default empty AI profiles appearing when the user had no saved profiles.
- Fixed several TypeScript inference issues.
- Fixed native browser alerts when selecting non-WebM formats on the web.

## Upgrade Notes

### API Keys Are Not Exported

AI profiles and API keys are stored only on the current device. Exported project files do not include API keys.

If you move to another device, you will need to create AI provider profiles again on that device.

### Web Video Export Limitation

The web version only exports WebM directly. Use the desktop app if you need MP4, MOV, or MKV.

### Choosing Full or Lite Desktop Builds

If you want one-click installation and video export that works out of the box, download `GalWriter-Setup.exe` Full. If download size matters more and FFmpeg is already installed on your computer, download `GalWriter-Setup-Lite.exe`.

### Windows Defender Warning

Windows Defender may warn about an unknown publisher because the app is not code-signed yet. Please verify that your download comes from the official GitHub Releases page before running it.

---

## Known Limitations

- This release mainly targets Windows users.
- Web video export only guarantees WebM.
- The Lite desktop app does not bundle FFmpeg, so non-WebM video formats depend on system FFmpeg.
- AI provider availability, quota, billing, and data policies depend on the provider configured by the user.
- Local Stable Diffusion WebUI requires users to start WebUI manually and enable its API.

## Download & Install

Download the Windows build from **Assets** on the GitHub Release page.

| File                                         | Description                                                                                 |
| -------------------------------------------- | ------------------------------------------------------------------------------------------- |
| `GalWriter-Setup.exe`                        | Full installer with slim bundled FFmpeg. Recommended for most users.                         |
| `GalWriter-Setup-Lite.exe`                   | Lite installer without bundled FFmpeg. Smaller download; non-WebM export needs system FFmpeg. |
| `GalWriter-AI-v1.2.5-windows-x64.exe`        | Portable Lite build. No installation required; non-WebM export needs system FFmpeg.          |
| `GalWriter-AI-v1.2.5-windows-x64-portable-full.zip` | Portable Full build. Extract and keep `ffmpeg.exe` next to the app executable.       |
| `Source code (zip)` / `Source code (tar.gz)` | Auto-generated GitHub source archives. These are not the app installer.                      |

If you only want to use the app, download an `.exe` file or portable zip instead of the Source code archives. Beginners and users who need MP4 / MOV / MKV export should download the Full build first.
