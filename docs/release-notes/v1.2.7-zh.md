# GalWriter AI v1.2.7

> 与 v1.2.5 相比，本次版本已经不只是一次小修补，而是把项目保存、AI 配置、移动端适配、Android 打包、界面风格和若干关键节点逻辑一起做了升级。它仍然重点修复了打开项目后保存键误高亮、自动保存与手动保存状态不同步、项目恢复提示过于打扰等问题，但整体上更像一次面向正式体验的综合更新。

**[English →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-en.md) | [日本語 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-ja.md)**

## 概览

GalWriter AI v1.2.7 是面向正式上线的一次综合升级。与 v1.2.5 的“本地项目库 + AI 配置 + 视频导出”基础相比，这一版继续扩展了移动端适配、Android 支持、AI 配置重构、API 指引、背景去除接口、UI 风格和项目状态管理，并把多媒体导出从外置 FFmpeg 组合包推进到更轻量的一体化方案，让编辑器更接近一个可以长期使用的本地创作工作台。

这一版最大的变化，是把项目快照、手动保存、自动保存和恢复流程重新对齐，同时把 1.2.5 之后积累的一批用户可感知改动一起收拢到正式发布口径里。打开已有项目后，保存按钮不应再无缘无故高亮；未发生修改时，也不会错误触发 autosave 或恢复提示。与此同时，项目标题、显示名和保存文件名的职责边界也被重新整理，避免刚打开项目时就出现“看起来像改过”的状态错觉。

---

## 亮点

- **打开项目不再误亮保存键**：刚进入已有项目时，保存按钮会保持正常状态，只有真正发生修改后才会进入“未保存”状态。
- **自动保存状态更稳定**：autosave 现在和手动保存使用同一套快照口径，减少“内容没变但状态变了”的情况。
- **恢复提示更克制**：只有在确实存在未保存草稿时，才会提示恢复上次进度。
- **项目标题与显示名职责更清晰**：项目标题不再被文件名反向污染，显示名与保存名的关系更明确。
- **关闭流程更完整**：在桌面端关闭应用时，会先处理未保存状态，减少误关或误丢内容的风险。
- **导入导出和保存链路更稳**：围绕项目序列化、恢复、保存回写做了更一致的处理。
- **多媒体导出更轻量**：从外置 FFmpeg 组合包升级为一体化方案，发布包和导出链路都更简洁。
- **AI 配置中心更完整**：文本、图片、去背景、语音的配置管理进一步统一，多个 Provider profile 的组织方式更清楚。
- **移动端与 Android 更成熟**：界面和交互对手机屏幕做了更多适配，Android 打包也进入可交付状态。
- **界面视觉更统一**：整体配色、按钮状态和部分节点外观做了同步更新。
- **背景去除与 API 指引更新**：AI 接口接入和使用说明更贴近当前支持的服务。

## 新增

### 更一致的项目状态同步

项目打开后，编辑器会先等待当前项目快照稳定，再把它当作“已保存基线”。这样可以避免因为恢复顺序、字段顺序或临时状态不同步，导致保存按钮一打开就高亮。

### 桌面端关闭确认

桌面端在存在未保存更改时，会优先进入应用内确认流程，避免直接关闭导致内容丢失。

### 移动端与 Android 适配

编辑器和相关面板继续针对手机屏幕做响应式调整，Android 版也从“可运行”推进到更适合移动预览与测试的状态。

### AI 配置重构

AI Profile 组织方式被重新梳理，文本、图片、去背景、语音的配置入口更统一，也更容易维护多个 Provider。

### 多媒体导出一体化

桌面端的多媒体导出不再依赖 v1.2.5 那种外置 FFmpeg 组合包，而是切换到更轻量的一体化方案，减少了发布包体积，也让导出链路更统一。

---

## 改进

### 保存与自动保存

手动保存和自动保存现在使用统一的快照比较口径，减少保存后仍显示脏状态的问题。

### 项目恢复逻辑

项目恢复后，系统会更谨慎地决定是否提示恢复上次进度，避免正常保存后的项目再次打开时被打断。

### 项目标题处理

项目标题、保存文件名和显示名之间的关系被重新整理，减少标题回填造成的状态误判。

### 接口与视觉更新

去背景 API 指引、部分 AI 接口说明和整体 UI 配色都做了更新，减少老说明和新能力不一致的情况。

### 多媒体导出轻量化

桌面端多媒体导出从外置 FFmpeg 组合包切换为一体化实现，发布资产更简洁，使用时也不再需要用户自己理解那套额外的 FFmpeg 配置方式。

---

## 修复

- 修复打开项目后保存按钮默认高亮的问题。
- 修复未修改内容时仍可能触发自动保存的问题。
- 修复正常保存后再次打开仍提示恢复上次进度的问题。
- 修复项目标题与文件名互相影响导致的状态错觉。
- 修复部分保存和恢复流程中的状态回写不一致问题。
- 修复部分移动端布局和手机视图上的显示问题。
- 修复数字判断节点中的紧急逻辑问题。
- 修复 AI 配置在部分场景下出现默认空配置的问题。
- 修复多媒体导出发布包过重、依赖说明过复杂的问题。

## 升级说明

### 这一版更适合正式体验

如果你是从旧版本升级到 v1.2.7，建议先用现有项目打开、保存一次，再重新打开确认状态正常。这样可以顺手把新的快照基线对齐到当前项目。

### 数据兼容

这一版仍然保持本地项目兼容优先的原则。已有项目可以继续打开和保存，不需要手工迁移。

### 1.2.5 之后的主要升级方向

如果把 v1.2.5 当作基础版本，那么 v1.2.7 这段时间的核心升级方向可以概括为：

- 更完整的 AI 配置中心
- 更成熟的移动端体验
- Android 打包与移动预览支持
- 更轻量的一体化多媒体导出
- 更统一的界面视觉
- 更稳的项目保存与恢复
- 更清晰的接口说明和兼容口径

## 已知限制

- 这一版仍主要面向 Windows 桌面体验。
