# GalWriter AI v1.2.7

> v1.2.5 から見ると、このリリースは単なる小修正ではありません。プロジェクト保存、AI Profile 管理、モバイル対応、Android パッケージング、見た目の整理、主要ノードの挙動までまとめて改善しつつ、プロジェクトを開いた直後に保存ボタンが誤って強調表示される問題などの状態同期問題も修正しています。

**[中文版 →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-zh.md) | [English →](https://github.com/Mingwen-Cui/GalWriter/blob/main/docs/release-notes/v1.2.7-en.md)**

## 概要

GalWriter AI v1.2.7 は、正式公開に向けたより広い意味での強化版です。v1.2.5 の「ローカルプロジェクト + AI Profile + 動画書き出し」を土台にしつつ、その後の更新でモバイル対応、Android サポート、AI Profile の再整理、API の案内更新、背景除去連携、UI の統一、プロジェクト状態管理の改善を進め、さらに動画や音声を含むマルチメディア書き出しを外部 FFmpeg 同梱型から、より軽量で一体化された方式へ移行し、長く使える制作ワークスペースに近づけました。

このバージョンの大きな変更は、プロジェクトスナップショット、手動保存、autosave、復元の流れを再度揃えたうえで、1.2.5 以降に積み上がったユーザー向けの改善をまとめて反映したことです。既存プロジェクトを開いた直後に保存ボタンが dirty 状態になることはなくなり、変更がない場合は autosave や復元確認も不要に出なくなります。プロジェクトタイトル、表示名、保存ファイル名の役割も整理し、読み込み直後から「何か変わった」ように見えないようにしました。

---

## ハイライト

- **プロジェクトを開いた直後に保存ボタンが誤強調されない**：既存プロジェクトを開いても、保存ボタンは通常状態のままです。
- **autosave がより安定**：autosave も手動保存と同じスナップショット基準を使うようになりました。
- **復元確認が控えめに**：未保存の autosave 下書きが本当にあるときだけ復元確認を表示します。
- **タイトルと表示名の関係を整理**：プロジェクトタイトルがファイル名に引っ張られなくなり、読み込み状態が分かりやすくなりました。
- **終了フローを改善**：デスクトップ版では未保存変更がある場合、終了前にアプリ内で確認します。
- **保存 / 復元フローを安定化**：シリアライズ、復元、保存への書き戻しをまとめて整えました。
- **マルチメディア書き出しを軽量化**：外部 FFmpeg 同梱型から一体化方式へ移行し、配布物と書き出し経路を簡潔にしました。
- **AI Profile センターを強化**：テキスト、画像、背景除去、音声の設定がより統一されました。
- **モバイル / Android 対応を拡充**：スマホ画面向けのレイアウト調整と Android 配布の準備が進みました。
- **見た目の統一感を改善**：配色やボタン状態、いくつかのノードの見た目を揃えました。
- **API 案内を更新**：背景除去や AI 利用の案内を現在の実装に合わせました。

## 追加

### より一貫したプロジェクト状態同期

プロジェクトを開いたあと、現在のスナップショットが安定するまで待ってから保存基準として扱うようにしました。これにより、復元順序やフィールド順、途中状態の違いで dirty 判定される問題を防ぎます。

### デスクトップ版の終了確認

デスクトップ版では未保存変更がある場合、まずアプリ内の確認フローを通すようにしました。これにより、誤終了によるデータ損失を抑えます。

### モバイル / Android 対応

スマホ向けの画面や操作感をさらに調整し、Android ビルドも配布対象として扱える段階に近づけました。

### AI Profile の再整理

Text / Image / Background-removal / Voice の設定をより統一的に扱えるようにし、複数 Provider の管理が分かりやすくなりました。

### マルチメディア書き出しの一体化

デスクトップ版のマルチメディア書き出しは、v1.2.5 のような外部 FFmpeg 同梱型ではなく、より軽量で一体化された方式に切り替わりました。配布サイズが減り、利用時の手順も分かりやすくなっています。

## 変更

### 保存と autosave

手動保存と autosave のスナップショット比較を統一し、保存後に dirty 状態が残るケースを減らしました。

### プロジェクト復元

復元確認は、正常に保存されたプロジェクトに対して不要に繰り返し出ないよう、より慎重に表示されます。

### プロジェクトタイトル処理

プロジェクトタイトル、保存ファイル名、表示名の役割を整理し、読み込み時の状態混乱を減らしました。

### API と見た目の更新

背景除去 API の案内、関連する AI 利用の説明、全体の UI 配色と状態表現を更新しました。

### 書き出し配布物の整理

デスクトップ版のマルチメディア書き出しは一体化され、FFmpeg を別途意識する必要が少なくなりました。

---

## 修正

- プロジェクトを開いた直後に保存ボタンが高亮する問題を修正。
- 変更がないのに autosave が発火する問題を修正。
- 正常保存後に再オープンすると復元確認が出る問題を修正。
- タイトルとファイル名の相互作用で、読み込み直後のプロジェクトが編集済みに見える問題を修正。
- 保存と復元の一部で状態の書き戻しが不一致になる問題を修正。
- モバイルレイアウトやスマホ表示のいくつかの問題を修正。
- 数字判断ノードの緊急ロジック問題を修正。
- AI 設定が空のときに不要なデフォルト項目が出る問題を修正。
- マルチメディア書き出し用の配布物が過剰に分かれていた問題を修正。

## アップグレードメモ

### 公開体験向けに適した版

旧バージョンから更新する場合は、既存プロジェクトを一度開いて保存し、その後もう一度開いて状態が正常にそろうことを確認すると安心です。

### データ互換性

このリリースでもローカルファーストの互換性方針は維持しています。既存プロジェクトはそのまま開いて保存できます。

### v1.2.5 以降の主な拡張

v1.2.5 をベースにしたとき、v1.2.7 までの主な拡張は次の通りです。

- より充実した AI Profile 管理
- さらに進んだモバイル UX
- Android パッケージング対応
- より軽量なマルチメディア書き出し
- より統一された UI 見た目
- より安定した保存 / 復元
- 更新された API 案内と互換性の整理

## 既知の制限

- このリリースは引き続き Windows デスクトップ体験を主対象としています。
- Web 版の動画書き出しはブラウザ制約の影響を受けます。
- AI Provider の利用可否、クォータ、課金、ポリシーは設定先サービスに依存します。
- ローカル Stable Diffusion WebUI はユーザーが起動し、API を有効化する必要があります。
